## 1. ローカル開発環境セットアップ

### 1.1 前提条件

- Docker Desktop（Windows）
- Git
- Python 3.11+（バックエンド開発時のみ）
- Node.js 20+（フロントエンド開発時のみ）

### 1.2 初回セットアップ

```bash
# リポジトリクローン
git clone https://github.com/[org]/construction-manager-v3.git
cd construction-manager-v3

# 環境変数設定
cp .env.example .env
# .env を編集（GEMINI_API_KEY を設定）
```

**.env の必須項目**:
```env
POSTGRES_DB=cmv3
POSTGRES_USER=cmv3user
POSTGRES_PASSWORD=（任意の安全なパスワード）
DATABASE_URL=postgresql+asyncpg://cmv3user:${POSTGRES_PASSWORD}@postgres:5432/cmv3
REDIS_URL=redis://cmv3-redis:6379/0
GEMINI_API_KEY=（Google AI Studio で取得）
JWT_SECRET=（32文字以上のランダム文字列）
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
JWT_REFRESH_TOKEN_EXPIRE_DAYS=7
ALLOWED_ORIGINS=http://localhost:3000
NEXT_PUBLIC_API_URL=http://localhost:8000
APP_ENV=development
LOG_LEVEL=DEBUG
```

### 1.3 開発環境起動

```bash
# 全サービス起動
docker compose -f docker-compose.dev.yml up -d

# 起動確認
curl http://localhost:8000/api/v1/health  # → {"status":"ok"}
curl http://localhost:3000               # → ログインページ
```

**開発環境サービス**:
| サービス | ポート | 説明 |
|---|---|---|
| cmv3-api | 8000 | FastAPI（uvicorn --reload）|
| cmv3-web | 3000 | Next.js（npm run dev）|
| cmv3-worker | - | Celery ワーカー |
| postgres | 5432 | PostgreSQL 16 |
| cmv3-redis | 6379 | Redis 7 |
| adminer | 8080 | DB管理UI（開発のみ）|

### 1.4 DBマイグレーション

```bash
# コンテナ内でマイグレーション実行
docker exec cmv3-api sh -c "cd /app && uv run alembic upgrade head"

# 現在のリビジョン確認
docker exec cmv3-api sh -c "cd /app && uv run alembic current"

# 新しいマイグレーション作成（スキーマ変更後）
docker exec cmv3-api sh -c "cd /app && uv run alembic revision --autogenerate -m 'add_new_feature'"
```

**マイグレーション規則**:
- ファイル名: `{8文字の英数字revision}_{機能名}.py`
- 必ず `upgrade()` と `downgrade()` を実装
- PostgreSQL ENUM の追加は `CREATE TYPE ... IF NOT EXISTS` を使用
- 既存テーブルへのカラム追加は `op.add_column()` を使用

### 1.5 シードデータ投入

```bash
docker exec cmv3-api sh -c "cd /app && uv run python app/scripts/seed.py"
# 投入内容:
# - 管理者ユーザー (hisa1975@gmail.com / admin1234)
# - 印紙税テーブル 8行
# - サンプル業者 3社
```

## 3. Docker Compose 設定（本番）

### docker-compose.prod.yml

```yaml
version: "3.9"

services:
  cmv3-api:
    build:
      context: ./backend
      dockerfile: Dockerfile
    restart: unless-stopped
    command: >
      sh -c "uv run alembic upgrade head &&
             uv run uvicorn app.main:app --host 0.0.0.0 --port 8000"
    environment:
      DATABASE_URL: ${DATABASE_URL}
      REDIS_URL: ${REDIS_URL}
      GEMINI_API_KEY: ${GEMINI_API_KEY}
      JWT_SECRET: ${JWT_SECRET}
      JWT_ACCESS_TOKEN_EXPIRE_MINUTES: ${JWT_ACCESS_TOKEN_EXPIRE_MINUTES}
      JWT_REFRESH_TOKEN_EXPIRE_DAYS: ${JWT_REFRESH_TOKEN_EXPIRE_DAYS}
      ALLOWED_ORIGINS: ${ALLOWED_ORIGINS}
      APP_ENV: production
      LOG_LEVEL: INFO
    volumes:
      - scan_uploads:/app/uploads
    networks:
      - cmv3-prod-network
    depends_on:
      cmv3-redis:
        condition: service_healthy

  cmv3-worker:
    build:
      context: ./backend
      dockerfile: Dockerfile
    restart: unless-stopped
    command: uv run celery -A app.tasks.celery_app worker --loglevel=info --concurrency=2
    environment: *cmv3-api-env  # cmv3-api と同じ環境変数
    volumes:
      - scan_uploads:/app/uploads
    networks:
      - cmv3-prod-network

  cmv3-web:
    build:
      context: ./frontend
      dockerfile: Dockerfile.prod
    restart: unless-stopped
    environment:
      NEXT_PUBLIC_API_URL: ${NEXT_PUBLIC_API_URL}
    networks:
      - cmv3-prod-network

  cmv3-nginx:
    image: nginx:alpine
    restart: unless-stopped
    ports:
      - "8005:80"
    volumes:
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf
    networks:
      - cmv3-prod-network

  cmv3-redis:
    image: redis:7-alpine
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - cmv3-prod-network

volumes:
  scan_uploads:

networks:
  cmv3-prod-network:
    external: false
```

### nginx/default.conf

```nginx
upstream api {
    server cmv3-api:8000;
}
upstream web {
    server cmv3-web:3000;
}

server {
    listen 80;
    client_max_body_size 50M;

    location /api/ {
        proxy_pass http://api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 120s;
    }

    location / {
        proxy_pass http://web;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 4. バックエンド実装規則

### 4.1 ディレクトリ規約

```
backend/app/
├── modules/{domain}/    # ビジネスロジック本体
│   ├── __init__.py
│   └── {feature}_router.py
├── models/{entity}.py   # SQLAlchemy モデル
├── schemas/{entity}.py  # Pydantic スキーマ（再利用率低いもの）
├── services/{service}.py # 共通サービス
└── shared/              # クロスモジュール利用
    ├── models/          # 共通モデル（enums, base, history）
    └── services/        # 共通サービス（quote_init 等）
```

### 4.2 ルーター登録規則

全ルーターは `app/main.py` の `include_router` で登録。プレフィックスは `/api/v1`。

```python
# main.py のパターン
from app.modules.project.ledger_router import router as ledger_router
app.include_router(ledger_router, prefix="/api/v1")
```

### 4.3 非同期 DB アクセス

```python
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from sqlalchemy.orm import selectinload

# リレーション付きクエリ
result = await db.execute(
    select(Project)
    .options(selectinload(Project.ledger_approvals))
    .where(Project.id == project_id)
)
project = result.scalar_one_or_none()
```

### 4.4 Celery タスク（AI解析）

```python
# tasks/scan_tasks.py
# asyncpg + Celery は MissingGreenlet エラーが発生するため
# psycopg2（同期）を使用すること

from sqlalchemy import create_engine  # 非同期ではない
```

### 4.5 編集履歴の自動記録

案件ステータス変更時など重要な操作は `edit_histories` に記録:
```python
from app.services.history import record_history

await record_history(
    db=db,
    project_id=project_id,
    entity_type="project",
    entity_id=project_id,
    change_type="update",
    changed_by=current_user.id,
    field_changes={"status": {"before": "quote", "after": "ordered"}}
)
```

## 5. フロントエンド実装規則

### 5.1 API 呼び出し

```tsx
// 必ず apiFetch を使うこと（自動リフレッシュ付き）
const data = await apiFetch<ProjectDetail>(`/api/v1/projects/${projectId}`);

// NG: fetch を直接使うのは PDF/Excel ダウンロード時のみ
const r = await fetch(`${API_URL}/api/v1/.../export-pdf`, {
  headers: { Authorization: `Bearer ${getToken()}` }
});
```

### 5.2 エラーハンドリング

```tsx
try {
  await apiFetch(...);
  showMsg("保存しました");
} catch (e) {
  showMsg(`エラー: ${(e as Error).message}`);
}
```

### 5.3 金額・日付の表示

```tsx
// 金額: fmtYen を使う（直接 toLocaleString() は使わない）
fmtYen(order.total_amount)  // → "¥1,240,000"

// 日付: fmtDateISO / fmtDateJP を使う
fmtDateISO(order.issue_date)  // → "2026-05-08"
```

### 5.4 ページ内状態管理

- サーバー状態: `useState` + `apiFetch`（SWR/TanStack Query は不使用）
- 保存後は必ず `await load()` で再取得
- メッセージ表示: `showMsg(text)` → 3〜4秒後に自動クリア

## 8. トラブルシューティング

### 8.1 よくある問題

**502 Bad Gateway（Nginx→コンテナ）**
```bash
# コンテナ再作成後はNginxのupstreamIPが変わる
docker restart cmv3-nginx
```

**マイグレーション失敗（MultipleHeadsError）**
```bash
# ブランチが分岐した場合
docker exec cmv3-api sh -c "cd /app && uv run alembic heads"
# → 2つの head が表示されたらマージが必要
uv run alembic merge -m "merge_heads" {head1} {head2}
```

**Celery タスク MissingGreenlet エラー**
```
原因: asyncpg（非同期）を Celery タスク内で使用
解決: psycopg2（同期）エンジンに変更（scan_tasks.py 参照）
```

**フロントエンドビルドエラー（TypeScript）**
```bash
# ローカルで型チェック（node_modules が必要）
cd frontend && npm install && npx tsc --noEmit
```

**ファイアウォール SSH 不通**
```
WebARENA Indigo のファイアウォール設定で自分のIPを許可する
管理コンソール: https://cp.webarena.ne.jp/
```
