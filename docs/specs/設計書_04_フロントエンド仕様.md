## 2. レイアウト構成

### 2.1 AppShell（全認証済みページ共通）

```
┌─ sidebar (240px) ─────┬─ main ──────────────────────────────┐
│ CL ロゴ + ブランド     │ [topbar: breadcrumbs + action]      │
│ ─────────────          │ [proj-header: サブナビ（案件配下）]  │
│ nav-items...           │                                     │
│  ダッシュボード         │  <children> (page content)          │
│  案件一覧              │                                     │
│  業者見積スキャン       │                                     │
│  承認待ち              │                                     │
│  ...                  │                                     │
│ ─────────────          │                                     │
│ [avatar] ユーザー名    │                                     │
└───────────────────────┴─────────────────────────────────────┘
```

**AppShell Props**:
```tsx
<AppShell
  breadcrumbs={[{ label: "案件一覧", href: "/projects" }, { label: "注文書" }]}
  action={<div>...</div>}  // トップバー右側のボタン群
>
  {children}
</AppShell>
```

### 2.2 案件サブナビゲーション（ProjectSubNav）

案件配下のページでは `app/projects/[id]/layout.tsx` が自動的にサブナビを挿入。

**タブ順序**:
```
詳細 | 工事台帳 | QCDS | 業者見積[n] | 顧客見積[n] | 注文書[n]
| 注文請書[n] | 請求書[n] | 進捗[n] | 工程表 | 出面 | 写真台帳
| 発注書 | 編集履歴[n]
```

バッジ `[n]` は件数を表示（`ProjectCounts` から取得）。0件のタブは `.zero` スタイル（薄く表示）。

## 4. 主要コンポーネント仕様

### 4.1 注文書・注文請書の共通デザインパターン

`order.html` 準拠。以下のパターンを注文書・注文請書ページで共用:

#### `.ord-hero`（ヘッダーカード）
```
display: grid; grid-template-columns: auto 1fr auto;
[番号チップ] | [タイトルh1 + 小見出しsmall] | [アクション群]
```

#### `.ord-status`（ステータスバッジ）
```
ドット + ラベル + ▾ キャレット
クリックでドロップダウン表示
各ステータスに border/bg/dot/text カラー対応
```

#### `.ord-timeline`（進捗ステップバー）
```
注文書: 下書き → 送付済 → 先方押印 → 受領済（4ステップ）
注文請書: 下書き → 発行済み（2ステップ）
done=緑チェック / cur=orange/purple 丸 / future=番号グレー
```

#### `.ord-field-grid`（発注先情報グリッド）
```
display: grid; grid-template-columns: 100px 1fr 100px 1fr;
.k セル: 灰ラベル（background: surface-2）
.v セル: 値（クリックでインライン編集）
.span2: grid-column: 2 / span 3
border-top/bottom/right で区切り
```

#### `.totals-card`（金額カード）
```
.head:  background: var(--c-primary); color: #fff; 大きな金額表示
.row:   flex justify-between; 各行
.major: background: surface-2; 強調行
.stamp-tax: background: warn 6%; 印紙税エリア
```

### 4.2 工事台帳ページ固有コンポーネント

#### ManualField（手動入力フィールド）
```
- 未入力: 薄黄背景 + 点線ボーダー + "✎ クリックして入力"
- 入力済: 白背景 + クリックで編集モード
- 編集中: Input/textarea + 保存/キャンセルボタン
```

#### ApprovalStamp（承認スタンプ枠）
4枠（社長・建築部長・経理・担当）それぞれ:
```
- 未依頼:  点線枠 + "押印依頼" ボタン → クリックでユーザー選択ドロップダウン
- 依頼済:  primary ボーダー + 依頼先ユーザー名 + "承認待ち" + 押印ボタン（依頼先のみ）
- 押印済:  success ボーダー + 押印者名 + 日付 + ×取消ボタン
```

#### LedgerWorkTable（表4統合テーブル）
```
列: No | 支払先 | 工種 | 実行予算[青] | 取決金額✎[橙] | ✓ | 取決差額[橙]
    | {開始月から6ヶ月}[緑] | 支払計[緑] | 残支払

- 取決金額: セルクリックでインライン number input
- ✓: HTMLチェックボックス → onChange で API PATCH
- 月別: セルクリックでインライン number input
- 支払計: 今月以前の月のみ合計（isPastOrCurrent関数）
- 支払開始月: テーブル上部のドロップダウンで選択（4月〜3月）
```

### 4.3 顧客見積書ページ（quote/[quote_id]/page.tsx）

```
2カラムレイアウト: grid-template-columns: 2.3fr 1fr

左カラム:
  - 見積ヘッダーカード（見積番号・発行日・有効期限・工期・支払条件）
  - 大項目セクション一覧（SectionBlock コンポーネント）
    - 各セクションに明細行テーブル（ItemRow コンポーネント）
  - 見積条件書カード（テンプレート選択・テキスト編集）

右カラム:
  - 承認スタンプ3枠（担当・査閲・承認）
  - QuoteTotals（合計カード: 御見積金額（税込）・内訳）
  - 承認依頼ボタン → モーダル（3ステップ選択）
  - ApprovalStamps（承認状態表示）
```

### 4.4 QCDSページ（qcds/page.tsx）

```
大型複合コンポーネント（1000行超）

セクション:
1. KPIストリップ（直接工事費・経費・営業利益・工事価格）
2. 工事割出サマリー（3列: 実行予算・取決見通・精算見通）
3. QCDSDirectWorkTable（直接工事費30行テーブル）
   - 外注/資材/その他の3カテゴリ
   - スキャン行のアコーディオン展開（▼/▶ ボタン）
4. QCDSExpensePanel（現場経費行: B-1 保険料・B-2 固定費）
5. 経費率設定パネル
```

## 5. 状態管理・データフェッチ

### 5.1 認証コンテキスト（AuthContext）

```tsx
// contexts/auth-context.tsx
const { user, login, logout, refreshUser } = useAuth();
// user: { id, email, full_name, role, department }
// localStorage に access_token / refresh_token を保存
// 401 レスポンス時に自動リフレッシュ
```

### 5.2 案件コンテキスト（ProjectSubNavContext）

```tsx
// contexts/project-context.tsx
const { project, counts } = useProjectSubNav();
// app/projects/[id]/layout.tsx がフェッチして提供
// ProjectSubNav がこれを読んでタブバッジ件数を表示
```

### 5.3 API クライアント（lib/api-client.ts）

```tsx
// 全 API 呼び出しはこれを通す
async function apiFetch<T>(path: string, options?: RequestInit): Promise<T>
// - 自動でアクセストークンを添付
// - 401 時にリフレッシュトークンで再試行
// - リフレッシュ失敗時に /login にリダイレクト
```

### 5.4 フォーマット関数（lib/format.ts）

```tsx
fmtYen(v)       // → "¥1,234,567" または "—"
fmtDateISO(s)   // → "2026-05-01" または "—"
fmtDateJP(s)    // → "2026年5月1日" または "—"
fmtDateTime(s)  // → "2026/05/01 14:30"
fmtNum(v)       // → "1,234,567" または "—"
fmtFileSize(b)  // → "1.2 MB"
```
