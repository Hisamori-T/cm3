### 1.4 取決金額 ↔ 発注書自動連動

- `qcds_direct_works.agreed_amount` = 同一業者（`vendor_id`）の発注書合計
- 連動タイミング: 発注書作成・更新・削除時にバックエンドで自動計算
- **分割発注対応**: 同一業者への複数発注書の合計金額を取決金額に反映
- 手動上書きも許可（連動値を上書きした場合はアイコンで表示）

```python
# services/qcds_sync.py
async def sync_agreed_amount_from_orders(db, project_id, vendor_id):
    total = await db.scalar(
        select(func.sum(PurchaseOrder.total_amount))
        .where(PurchaseOrder.project_id == project_id)
        .where(PurchaseOrder.vendor_id == vendor_id)
        .where(PurchaseOrder.status != 'cancelled')
    )
    await db.execute(
        update(QCDSDirectWork)
        .where(QCDSDirectWork.qcds_id.in_(...))
        .where(QCDSDirectWork.vendor_id == vendor_id)
        .values(agreed_amount=total or 0)
    )
```

## 2. 権限ロール 複数選択対応

### 2.1 DBスキーマ変更

```sql
-- users テーブルに roles 配列カラム追加
ALTER TABLE users ADD COLUMN roles userrole[] NOT NULL DEFAULT '{member}'::userrole[];

-- 既存データ移行: role → roles（単一要素配列）
UPDATE users SET roles = ARRAY[role]::userrole[];

-- インデックス（GINインデックスで高速検索）
CREATE INDEX ix_users_roles ON users USING GIN(roles);
```

`role` カラムは表示用の「主要ロール」として維持（廃止しない）。

### 2.2 Backend 権限チェック更新

```python
# shared/services/permissions.py
def has_role(user: UserModel, *required_roles: UserRole) -> bool:
    """ユーザーが指定ロールのうち少なくとも1つを持つか確認"""
    return any(r in user.roles for r in required_roles)

def require_roles(user: UserModel, *required_roles: UserRole):
    """指定ロールを持たない場合 403 を送出"""
    if not has_role(user, *required_roles):
        raise HTTPException(status_code=403, detail="権限がありません")
```

既存の `user.role == UserRole.admin` を `has_role(user, UserRole.admin, UserRole.super_admin)` に順次置換。

### 2.3 API スキーマ変更

```python
# schemas/user.py
class UserCreate(BaseModel):
    roles: list[UserRole] = [UserRole.member]  # 複数選択
    role: UserRole = UserRole.member            # 主要ロール（表示用）

class UserUpdate(BaseModel):
    roles: list[UserRole] | None = None
    role: UserRole | None = None
```

### 2.4 Frontend ユーザー管理 UI

- ロール選択: 単一 `<select>` → **チェックボックスグループ**
- 全ロールを表示し、複数チェック可能
- 主要ロール（表示用）: ラジオボタンで1つ選択
- ロールバッジ: ユーザー一覧に複数バッジ表示（最大3個、超過は "+N" 省略）

## 8. 請求書 — 総額請求書＋分割連動型（2026-06-06 承認）

### 8.1 概要

現行の「分割請求書 n 枚」方式を廃止し、
**総額請求書（親）＋分割請求書 1〜n 枚（子）** の親子構造に変更する。  
入金管理は総額請求書のみで行い、各分割への自動連動を実装する。

### 8.2 DBスキーマ変更

```sql
-- migration: invoice_split_v2

ALTER TABLE invoices
  ADD COLUMN invoice_type VARCHAR(20) NOT NULL DEFAULT 'standalone',
  -- 'standalone' | 'total' | 'split'
  ADD COLUMN parent_invoice_id UUID REFERENCES invoices(id) ON DELETE SET NULL;

ALTER TABLE payments
  ADD COLUMN target_split_id UUID REFERENCES invoices(id) ON DELETE SET NULL;
  -- どの回（split）の入金かを記録
```

### 8.3 請求書タイプ定義

| `invoice_type` | 説明 | 入金記録 | 削除可否 |
|---|---|---|---|
| `standalone` | 既存の通常請求書（変更なし）| ○ | 入金済以外 |
| `total` | 総額請求書（分割の親）| ○ **のみここで記録** | 子が全て未入金の場合 |
| `split` | 分割請求書（子、n 枚）| × 表示のみ | 総額請求書から自動更新 |

### 8.4 自動分割フロー

```
1. 請求書詳細で billing_method=percentage, billing_percentage=25 を設定・保存
2. 「自動分割」ボタン押下
3. API: POST /invoices/{id}/auto-split（既存エンドポイントを拡張）
   └── 現在の請求書 → invoice_type='total' に変換
       total_amount = 顧客見積 subtotal（税込）
   └── split 1〜4 を作成（parent_invoice_id = 総額ID）
       各 current_purchase = floor(subtotal_excl × pct / 100)
       最終回は端数込み
4. 一覧にて: 総額請求書を親行、分割 1〜n を折りたたみ子行で表示
```

### 8.5 入金記録フロー（total 請求書のみ）

**入金フォーム項目**:
- 入金日 *
- 金額（円）*
- **対象回** ← 新規: 「第1回（¥XXX）」「第2回（¥XXX）」ドロップダウン
- 支払方法
- 備考

**保存後の自動連動処理**:

```python
def on_payment_saved(payment):
    split_inv = payment.target_split  # 対象の分割請求書
    split_inv.status = 'paid' if payment.amount >= split_inv.total_amount else 'partially_paid'

    next_split = get_next_split(split_inv)  # 次の回の分割請求書
    if next_split:
        next_split.previous_balance = split_inv.total_amount
        next_split.received_amount  = payment.amount
        # 未払い分を次回に繰り越し
        unpaid = split_inv.total_amount - payment.amount
        next_split.current_purchase = next_split.billing_split_amount + unpaid
        _calc_totals(next_split)
```

**入金なし（前払なし）の場合**:  
次回の `current_purchase` = 前回の `billing_split_amount`（今月分）+ 前回の未払額

# modules/report/routers/invoices.py
def _calc_totals(inv: Invoice, extra_amount: float = 0.0) -> None:
    purchase = float(inv.current_purchase or 0) + extra_amount
    inv.outstanding_balance = float(inv.previous_balance or 0) - float(inv.received_amount or 0)
    inv.tax_amount = floor(purchase * 0.10)
    inv.total_amount = purchase + floor(purchase * 0.10)

### 10.6 Pydantic スキーマ（確定版）

> **型定義方針（2026-06-08 確定）**  
> - 金額フィールド（円・整数）: `float` 禁止 → **`int`** を使用  
>   理由: `Numeric(12,0)` は小数なし。浮動小数点誤差を排除（`10_000_000 * 0.005 = 50000.00000000001`）
> - 率フィールド（0.0〜1.0）: **`Decimal`** を使用  
>   理由: `0.005`（0.5%）は IEEE754 で正確に表現できない
> - 既存コードの `float` 残存は別スプリントで一括修正（R-1 スコープ外）

#     project_role: str
#
#   ProjectDetail:
#     project_role: str
```

> **`Decimal` の JSON シリアライズ注意**  
> Pydantic v2 のデフォルトでは `Decimal` は JSON 文字列で出力される（例: `"0.0050"`）。

### 10.7 サービス層 — 控除計算ロジック

```python
# backend/app/modules/report/services/deduction_service.py

_DEDUCTION_LABEL_JA = {
    "safety_fee":         "安全協力会費",
    "materials_advance":  "材料費立替",
    "parking_fee":        "駐車場代",
    "statutory_welfare":  "法定福利費",
    "other":              "その他",
}

def calculate_deduction_amount(
    deduction_type: str,
    calculation_rate: float | None,
    amount: float | None,
    invoice_current_purchase: float,
) -> float:
    """控除金額を計算する。rate 指定時は自動計算、それ以外は amount を使用。"""
    if calculation_rate is not None:
        return round(invoice_current_purchase * calculation_rate)
    if amount is not None:
        return amount
    raise ValueError("amount または calculation_rate のどちらかが必要です")


def recalculate_invoice_deduction_totals(invoice: Invoice) -> None:
    """invoice_deductions の合計を再計算して Invoice に反映する。"""
    active = [d for d in invoice.deductions if not d.is_deleted]
    total_deduction = sum(d.amount for d in active)
    invoice.total_deduction_amount = total_deduction
    invoice.final_payable_amount   = float(invoice.total_amount or 0) - total_deduction
```

```python
"""控除項目サービス — 追加・更新・削除・合計再計算・進捗サマリー。"""
from __future__ import annotations

import uuid
from decimal import Decimal
from math import floor

import structlog
from fastapi import HTTPException, status
from sqlalchemy import func, select
from sqlalchemy.ext.asyncio import AsyncSession

from app.models.enums import EditHistoryChangeType, InvoiceStatus
from app.models.invoice import Invoice, InvoiceDeduction
from app.schemas.invoice import (
    InvoiceDeductionCreate, InvoiceDeductionRead,
    InvoiceDeductionUpdate, ProgressSummaryResponse,
)
from app.services.history import record as record_history
from app.shared.constants.deduction import DEDUCTION_LABEL_JA  # ← shared から import

logger = structlog.get_logger(__name__)


# ── ヘルパー ────────────────────────────────────────────────────────────────

async def _get_invoice_or_404(
    invoice_id: uuid.UUID,
    project_id: uuid.UUID,   # URL の project_id と invoice.project_id を照合
    db: AsyncSession,
) -> Invoice:
    inv = await db.get(Invoice, invoice_id)
    if inv is None or inv.project_id != project_id:
        raise HTTPException(status_code=404, detail="請求書が見つかりません")
    return inv


def _guard_draft(invoice: Invoice) -> None:
    """draft 以外では控除変更を拒否する。"""
    if invoice.status != InvoiceStatus.draft:
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail="発行済み / 支払済みの請求書は変更できません",
        )


def _calc_amount(
    body: InvoiceDeductionCreate | InvoiceDeductionUpdate,
    invoice: Invoice,
) -> int:
    """控除金額を確定する（rate 指定時は current_purchase［税抜］から自動計算）。
    ※計算基準は税抜の today の請求額。税込ベースにする場合は total_amount に変更。
    """
    if getattr(body, "calculation_rate", None) is not None:
        base = int(invoice.current_purchase or 0)   # 税抜ベース
        return floor(base * float(body.calculation_rate))
    if body.amount is not None:
        return int(body.amount)
    raise HTTPException(
        status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
        detail="amount または calculation_rate のどちらかが必要です",
    )


async def _recalculate_totals(invoice: Invoice, db: AsyncSession) -> None:
    """active な控除の合計を集計し、Invoice.total_deduction_amount / final_payable_amount を更新する。"""
    result = await db.execute(
        select(func.sum(InvoiceDeduction.amount))
        .where(
            InvoiceDeduction.invoice_id == invoice.id,
            InvoiceDeduction.is_deleted.is_(False),
        )
    )
    total_deduction = int(result.scalar() or 0)
    invoice.total_deduction_amount = total_deduction
    invoice.final_payable_amount   = int(invoice.total_amount or 0) - total_deduction
    logger.info("deduction_totals_updated",
                invoice_id=str(invoice.id),
                total_deduction=total_deduction,
                final_payable=invoice.final_payable_amount)


def _to_deduction_read(d: InvoiceDeduction) -> InvoiceDeductionRead:
    return InvoiceDeductionRead(
        id=d.id,
        invoice_id=d.invoice_id,
        deduction_type=d.deduction_type,
        description=d.description,   # None の場合フロントで DEDUCTION_LABEL を使用
        amount=int(d.amount),
        calculation_rate=d.calculation_rate,
        account_hint=d.account_hint,
        is_deleted=d.is_deleted,
        row_no=d.row_no,
        created_at=d.created_at,
    )


# ── 控除 追加 ───────────────────────────────────────────────────────────────

async def add_deduction(
    project_id: uuid.UUID,
    invoice_id: uuid.UUID,
    body: InvoiceDeductionCreate,
    db: AsyncSession,
    user_id: uuid.UUID,
) -> InvoiceDeductionRead:
    invoice = await _get_invoice_or_404(invoice_id, project_id, db)
    _guard_draft(invoice)

    amount = _calc_amount(body, invoice)

    # row_no: 未指定なら active 行の末尾 + 1
    if body.row_no is None:
        count_result = await db.execute(
            select(func.count(InvoiceDeduction.id))
            .where(InvoiceDeduction.invoice_id == invoice_id,
                   InvoiceDeduction.is_deleted.is_(False))
        )
        body_row_no = (count_result.scalar() or 0) + 1
    else:
        body_row_no = body.row_no

    deduction = InvoiceDeduction(
        invoice_id=invoice_id,
        deduction_type=body.deduction_type,
        description=body.description,
        amount=amount,
        calculation_rate=body.calculation_rate,
        row_no=body_row_no,
    )
    db.add(deduction)
    await db.flush()

    await _recalculate_totals(invoice, db)
    await record_history(
        db, entity_type="invoice", entity_id=invoice_id,
        change_type=EditHistoryChangeType.update,
        field_changes={"deductions": {
            "before": None,
            "after": {"type": body.deduction_type.value, "amount": amount},
        }},
        user_id=user_id,
    )
    await db.commit()
    await db.refresh(deduction)
    return _to_deduction_read(deduction)


# ── 控除 更新 ───────────────────────────────────────────────────────────────

async def update_deduction(
    project_id: uuid.UUID,
    invoice_id: uuid.UUID,
    deduction_id: uuid.UUID,
    body: InvoiceDeductionUpdate,
    db: AsyncSession,
    user_id: uuid.UUID,
) -> InvoiceDeductionRead:
    invoice = await _get_invoice_or_404(invoice_id, project_id, db)
    _guard_draft(invoice)

    deduction = await db.get(InvoiceDeduction, deduction_id)
    if deduction is None or deduction.invoice_id != invoice_id or deduction.is_deleted:
        raise HTTPException(status_code=404, detail="控除項目が見つかりません")

    if body.description is not None:
        deduction.description = body.description
    if body.row_no is not None:
        deduction.row_no = body.row_no
    if body.calculation_rate is not None or body.amount is not None:
        deduction.amount           = _calc_amount(body, invoice)
        deduction.calculation_rate = body.calculation_rate

    await _recalculate_totals(invoice, db)
    await db.commit()
    await db.refresh(deduction)
    return _to_deduction_read(deduction)


# ── 控除 削除（論理削除） ───────────────────────────────────────────────────

    project_id: uuid.UUID,
    invoice_id: uuid.UUID,
    deduction_id: uuid.UUID,
    db: AsyncSession,
    user_id: uuid.UUID,
) -> None:
    invoice = await _get_invoice_or_404(invoice_id, project_id, db)

    if deduction is None or deduction.invoice_id != invoice_id or deduction.is_deleted:
        raise HTTPException(status_code=404, detail="控除項目が見つかりません")

    before_snapshot = {"type": deduction.deduction_type.value, "amount": int(deduction.amount)}
    deduction.is_deleted = True

    await _recalculate_totals(invoice, db)
    await record_history(
        db, entity_type="invoice", entity_id=invoice_id,
        change_type=EditHistoryChangeType.update,
        field_changes={"deductions": {"before": before_snapshot, "after": None}},
        user_id=user_id,
    )
    await db.commit()


# ── 出来高サマリー（動的計算） ─────────────────────────────────────────────

async def get_progress_summary(
    project_id: uuid.UUID,
    db: AsyncSession,
) -> ProgressSummaryResponse:
    """案件の累計請求額・残高を動的に計算して返す。保存せず毎回集計する。"""
    # 確定済み（sent/paid）請求書の current_purchase 合計
    result = await db.execute(
        select(func.sum(Invoice.current_purchase))
        .where(
            Invoice.project_id == project_id,
            Invoice.status.in_([InvoiceStatus.sent, InvoiceStatus.paid]),
            Invoice.current_purchase.is_not(None),
        )
    )
    cumulative = int(result.scalar() or 0)

    # draft 請求書の current_purchase（最新1件）でスナップショットの contract を取得
    draft_result = await db.execute(
        select(Invoice.current_purchase, Invoice.contract_amount_snapshot)
        .where(Invoice.project_id == project_id, Invoice.status == InvoiceStatus.draft)
        .order_by(Invoice.created_at.desc())
        .limit(1)
    )
    draft_row = draft_result.first()
    current  = int(draft_row.current_purchase or 0) if draft_row else None
    contract = int(draft_row.contract_amount_snapshot or 0) if (draft_row and draft_row.contract_amount_snapshot) else None

    outstanding = (contract - cumulative - (current or 0)) if contract else None
    percent     = (Decimal(cumulative) / Decimal(contract) * 100).quantize(Decimal("0.1")) \
                  if contract else None

    return ProgressSummaryResponse(
        contract_amount=contract,
        cumulative_billed=cumulative,
        current_purchase=current,
        outstanding_contract=outstanding,
        progress_percent=percent,
    )
