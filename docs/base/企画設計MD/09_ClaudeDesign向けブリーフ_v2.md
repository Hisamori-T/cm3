## 2. 重要な追加コンテキスト

ひささんからのヒアリング回答（実装に影響する重要事項）：

1. **平和堂が約170店舗** ── 主要顧客で高頻度リピート。残りの顧客は年間数回
2. **工程表は紙ベース** ── デジタル化したいが、日/週/月単位を柔軟に切り替えたい
3. **日報文化なし** ── 今回を機に導入したいが、最初は任意提出
4. **スマホはLINE・電話のみ** ── 業務利用未開拓。LINE並みの簡単操作が必須
5. **現場↔事務所連絡は個別LINE** ── グループLINEではない。個人宛通知が現実的
6. **発注書は紙運用** ── デジタル化で効率改善余地大
7. **入金確認は手動** ── 銀行明細を見て請求書と突合している

これらを踏まえて、デザイン方針：

- **LINE並みの簡単操作を全モバイル画面で徹底**
- **平和堂170店舗を快適に扱えるUI**（店舗選択は検索＋頻出ピン留め）
- **「日報書くと得する」UI**（強制感ゼロ、ストリーク・いいね等）
- **紙派の人にも優しい**（PDF印刷ボタンを目立つ位置に）

### S19 工程表（案件単位）

サクミルのガントチャート（スクショ参照）を参考に、より洗練された形で：

**要件**：
- 横軸：日/週/月切替（クラップさんの「柔軟対応」要望に応える）

### S21 日報入力（スマホ）

**最重要画面**。日報文化のない会社に定着させるための核。

**UI原則**：
1. **3タップで提出完了** が理想（案件選択→時間→写真→OK）
2. **前日の日報を引き継ぎ**（同じ案件・工程ならコピペ感覚）
3. **入力欄は必要最小限**：案件、開始終了時刻、写真、一言メモのみ
4. **音声入力ボタンを目立つ位置に**
5. **完了後の達成感**：「お疲れ様！🎉 7日連続提出中🔥」

**画面構造**：

```
┌─────────────────────────┐
│ ← 日報　　　　　 [提出] │
├─────────────────────────┤
│ 📅 5月13日(水) ☀ 22°C │
├─────────────────────────┤
│ [+ 作業を追加]          │
│                         │
│ ┌─ 前日の作業を引継 ─┐ │
│ │ アル・プラザ アミ   │ │ ← タップで取込
│ │ 内装解体  8.5h     │ │
│ └─────────────────────┘ │
│                         │
│ ┌─ 今日の作業 ───────┐ │
│ │ ✕ 削除              │ │
│ │ 案件: アル・プラザ  │ │
│ │ 工程: 内装解体      │ │
│ │ ⏰ 8:30 - 17:00     │ │
│ │ 📷 [撮影]           │ │
│ │ 💬 [音声] [タイプ]  │ │
│ └─────────────────────┘ │
│                         │
│ ✏ 一言メモ             │
│ [音声で入力 🎤]         │
└─────────────────────────┘
```

**配色**：
- 提出ボタン：クラップネイビー、画面右上固定
- カメラ・音声入力：プライマリ強調色（ティール）
- 削除：薄いグレー（誤タップ防止）

## 6. 補足：日報UIに込めるべき思想

日報文化のない会社に日報を導入することの難しさを設計でカバー：

**やってはいけないこと**：
- 入力項目を細かく増やす（書く気が失せる）
- 「未提出」を赤く強調（罪悪感を煽る）
- 上司の管理ビューを強調（監視感が出る）
- 提出を必須にする（任意提出方針）

**やるべきこと**：
- 「書きたくなる」演出（いいね、ストリーク、コメント）
- 「書くと楽になる」効果（過去案件の引継、原価集計の自動化）
- 「同僚が書いてる」見える化（書かないと孤立感、ただしポジティブな圧）
- 1日空白でも責められない設計（連続記録は強調しても、未提出は淡く）
