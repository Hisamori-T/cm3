## トラブル時の対処

### Claude Code がスキーマを勝手に変える

`CLAUDE.md` の「してはいけないこと」を再度読ませる：
```
@CLAUDE.md を再読してください。特に「してはいけないこと」の
セクション。設計書から逸脱した実装をしないでください。
```

### 実装が長くなりすぎてClaude Codeが混乱

Stepを細かく区切る。1回のプロンプトで「1つの機能」だけ実装させる。

### Anthropic API のレート制限（HTTP 529）

ひささんが以前経験した overloaded_error。指数バックオフ：
- 1回目失敗 → 1秒待つ
- 2回目失敗 → 2秒待つ
- 3回目失敗 → 4秒待つ
- 4回目失敗 → 8秒待つ

### コンテキスト窓不足

設計書すべてを `@` で参照せず、必要な章だけ指定。例：
- ❌ `@docs/02_設計書_Part1.md` 全体
- ✅ `@docs/02_設計書_Part1.md の第2章だけ読んで`

## トークン節約のコツ（Claude Code利用時）

1. **CLAUDE.mdを充実させる** ← 毎回読まれるので、ここに重要情報を集約
2. **長い設計書は章単位で指定**
3. **完了後はClaude Codeセッションをこまめにリセット**
4. **同じ機能の修正は同じセッションで完結させる**（コンテキストが活きる）
5. **大きな変更は事前に「実装計画を提示」させ、レビューしてから実行**
