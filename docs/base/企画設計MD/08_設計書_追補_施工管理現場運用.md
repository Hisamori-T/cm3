## 0. このドキュメントの位置づけ

競合分析（07_競合分析）で、サクミルとプロワンの機能セットを Construction Manager v3 に取り入れる方針が決まった。ひささんからの追加情報：

- **工程表**：紙ベースで運用中。デジタル化できると良い
- **作業日報**：未導入。今回を機に始めたい
- **顧客リピート率**：平和堂が主要顧客で非常に高頻度、その他は年間数回程度
- **スマホ利用**：全員所有、電話・メール・LINE中心。業務利用は未開拓
- **方針**：他社アプリでできることは何でもできるようにしたい（AIチャットボットは除く）

これらを踏まえて、Phase 1B として組み込む施工管理・現場運用機能を設計する。

**重要な配慮**：
- 日報は文化として未導入なので、**抵抗なく定着するUI**が必須
- スマホは持っているが業務未利用なので、**LINE並みの簡単操作**を目指す
- 平和堂中心の取引なので、**顧客マスタは平和堂を起点に強化**する

### 3.4 日報→QCDS自動連動

日報の作業時間から、QCDSの「実際の現場人件費」を自動算出：

```python
def calculate_actual_labor_cost_from_reports(project_id: UUID, db: AsyncSession) -> Decimal:
    """日報から案件の実際の現場人件費を算出"""
    # 案件に紐づく全日報項目を取得
    entries = await get_daily_report_entries_by_project(project_id, db)

    total_cost = Decimal(0)
    for entry in entries:
        # 担当者の人件費単価を取得
        user = entry.daily_report.user
        unit_price = await get_user_labor_unit_price(user.id, db)  # 月単価
        # 月単価を時間単価に変換（月160h想定）
        hourly_rate = unit_price / Decimal(160)
        hours = Decimal(entry.working_minutes) / Decimal(60)
        total_cost += hourly_rate * hours

    return total_cost
```

日報を書くと自動的に「実際の現場人件費」が積み上がっていき、QCDSの計算に反映される。

```sql
CREATE VIEW vendor_attendance_summary AS
SELECT
    project_id,
    vendor_id,
    DATE_TRUNC('month', attendance_date) AS month,
    SUM(worker_count) AS total_worker_count,
    COUNT(DISTINCT attendance_date) AS working_days,
    SUM(amount) AS total_amount
FROM vendor_attendances
GROUP BY project_id, vendor_id, DATE_TRUNC('month', attendance_date);
```

### 10.4 LINE Notify 統合

```python
# backend/app/services/notification_service.py
async def send_line_notify(user_id: UUID, message: str, image_url: str = None):
    token = await get_user_line_token(user_id, db)
    if not token:
        return
    async with httpx.AsyncClient() as client:
        await client.post(
            "https://notify-api.line.me/api/notify",
            headers={"Authorization": f"Bearer {token}"},
            data={"message": message, "imageThumbnail": image_url, "imageFullsize": image_url},
        )
```

## 13. F-43：スマホ写真撮影→即アップロード

### 13.1 PWA カメラ統合

PWAでカメラアクセスして、撮影→即アップロードの最速フロー：

```javascript
// frontend/src/components/PhotoUploader.tsx
const PhotoUploader = ({ projectId, onUpload }) => {
  const handleCapture = async (e) => {
    const file = e.target.files[0];
    // GPS位置情報取得
    const position = await new Promise((resolve) => {
      navigator.geolocation.getCurrentPosition(resolve, () => resolve(null));
    });

    // FormData作成
    const formData = new FormData();
    formData.append('file', file);
    formData.append('projectId', projectId);
    if (position) {
      formData.append('lat', position.coords.latitude);
      formData.append('lng', position.coords.longitude);
    }
    formData.append('takenAt', new Date().toISOString());

    // アップロード（プログレス表示）
    await uploadWithProgress(formData);
    onUpload();
  };

  return (
    <label className="big-camera-button">
      <input type="file" accept="image/*" capture="environment" onChange={handleCapture} hidden />
      📷 撮影してアップロード
    </label>
  );
};
```

- `capture="environment"` でリアカメラ起動
- アップロード後に自動的に案件・日報に紐付け
- 失敗時はローカル一時保存→ネット復帰時に再送
