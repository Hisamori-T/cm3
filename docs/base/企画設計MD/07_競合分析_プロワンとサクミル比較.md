# 競合分析：プロワン vs サクミル vs Construction Manager v3

**作成者**：平等 久盛
**作成日**：2026年5月13日
**目的**：Construction Manager v3が「現行Excel以上の成果」を出すために、競合サービスから何を学び、何を取り入れるべきかを整理する

---

## 0. このドキュメントの位置づけ

ひささんから「プロワンとサクミルにできる機能はつけたい。いいとこどりして、現在のExcel表で管理している以上の成果を出せるようにするには何が足りないか」という要望があった。

両サービスを調査し、現状のCM-v3設計（企画書・設計書Part1/Part2・追補①承認フロー・追補②QCDS業務マスタ）と比較してギャップを抽出する。

**重要な前提**：
クラップさんの現行Excelは単なる工事台帳ではなく、QCDS原価算定表・印紙税自動算定・営業利益逆算など、競合サービスにも実装されていない高度な原価管理ロジックを既に持っている。つまり「Excel < CM-v3」になるだけでなく、「プロワン/サクミル < CM-v3」になる余地がある。

---

## 1. 各サービスの機能まとめ

### 1.1 プロワン（pro-one-cloud.com）

**位置づけ**：「現場から経営までつなぐオールインワン業務管理SaaS」。短期工事・リフォーム・設備工事系の業界特化型Vertical SaaS。

**主要機能**：

| カテゴリ | 機能 |
|---|---|
| 営業 | 顧客管理（CRM）、見込み客管理、営業フェーズ管理、営業目標可視化 |
| 案件 | 案件受領〜見積〜発注〜請求の一気通貫管理 |
| 現場 | スマホ/タブレットでの現場入力、写真撮影、作業報告、ルート最適化（移動距離最小化） |
| 承認 | 見積・発注・経費申請のデジタル承認フロー |
| 集計 | 売上・工事原価の自動集計、経営判断スピード向上 |
| 連携 | 集客プラットフォーム「ミツモア」とのリード連携、IT導入補助金対応 |
| AI | 電話応対の自動化、応対記録の自動要約 |
| その他 | 機材一つひとつの履歴管理（機材CRM）、紙・Excel・複数システムからの脱却 |

**特徴**：
- BtoBサービスとして既に多くの導入実績あり
- AI機能（電話応対自動化）が進んでいる
- リード獲得から商談管理まで含む「マーケティング寄り」の機能群
- 移動ルート最適化など、複数現場を回る業態向け

**価格**：要見積もり（公表なし）

### 1.2 サクミル（sakumiru.jp）

**位置づけ**：「従業員数100名未満の現場アプリ・業務支援クラウド」。建設業向け中小企業特化。月額9,800円/30アカウント（2025年9月改定）。

**主要機能**：

| カテゴリ | 機能 |
|---|---|
| 顧客 | 顧客・作業現場ごとの情報管理 |
| 案件 | 案件詳細・工程・ステータス管理 |
| ファイル | 案件ごとの写真・図面・各種ファイル保存 |
| スケジュール | 個人・現場の予定可視化、職人スケジュールと現場工程の連動 |
| **日報** | スマホで作業日報入力、現場で即提出、写真添付、リアルタイム共有 |
| **写真台帳** | Excelで写真選択することなく最小限の操作で写真台帳作成 |
| 見積・請求 | 見積書から請求書への自動変換、帳票作成・管理 |
| 原価・粗利 | スケジュール・作業日報との連動による資材費・労務費・協業費の自動集計 |
| **経営レポート** | 売上・粗利、案件や人ごとの稼働時間が自動表示されるダッシュボード |
| **工程表** | ガントチャート工程表、クラウド管理で即共有 |

**特徴**：
- 価格が圧倒的に安い（月額9,800円）
- スマホファースト、現場での入力が前提
- 「日報→原価集計」の自動連動が中核
- 工程表（ガントチャート）に強い
- App Store ★4.2、3,000社以上導入

### 1.3 現状のConstruction Manager v3 設計

**位置づけ**：株式会社クラップ専用、自社運用Web工事台帳。

**設計済機能**：

| カテゴリ | 機能 |
|---|---|
| 案件 | 工事台帳CRUD、自動採番、7段階ステータス、編集権限・履歴 |
| QCDS | 原価算定表、業務マスタによる自動計算（印紙税・保険・経費率・利益逆算） |
| 帳票 | 見積書・注文書・請求書のPDF/Excel出力、印紙税自動算定 |
| **承認** | 電子捺印・デジタル印影・編集による自動リセット |
| AI | Gemini APIによる業者見積スキャン解析、QCDS/見積への自動転記 |
| マスタ | 業者マスタ、過去単価履歴、見積流用 |
| Excel連携 | インポート/エクスポート双方向 |
| 進捗 | 写真・図面アップロード、テキストログ |
| 集計 | 全案件横断ダッシュボード、ステータス分布、月別推移、利益率ランキング |

## 3. CM-v3 が持っていて競合にない強み

これらは絶対に手放さず、強みとして打ち出す：

1. **QCDS高度な原価算定** — 競合は原価管理機能はあるが、クラップ社のQCDSのような事業部経費率・保険料率・人件費単価まで組み込んだ精緻な仕組みではない
2. **印紙税の自動算定（契約・領収書両方）** — 法的書類対応として競合より一歩進んでいる
3. **営業利益逆算テーブル** — 工事原価から複数の利益率パターンで顧客提示価格を自動算出
4. **AI見積解析（Gemini Vision）** — 業者見積をスキャンしてQCDS・見積に自動転記。プロワンの「電話応対AI」とは方向性が違うが、原価系業務に絞った深いAI活用
5. **承認フロー＋デジタル印影** — 見積書のPDF/Excelに承認結果が印影として自動反映されるレベルは競合にない
6. **編集による承認自動リセット＋スナップショットハッシュ** — 競合にはない厳密な承認管理
7. **Excel双方向連携** — 高齢従業員のExcel派が並行運用できる移行配慮
8. **編集権限制御＋詳細な編集履歴** — 監査性が高い
9. **業者見積の単価履歴蓄積＋流用** — 単なる業者マスタを超えた知識資産化

## 4. CM-v3 に足りない機能（取り入れるべき機能）

サクミル・プロワンを参考に、Phase 1〜2で追加すべき機能を整理：

### 4.1 【最優先】Phase 1に組み込むべき機能

#### A. ガントチャート工程表

サクミルの中核機能。現状CM-v3は「工期（見積/契約/実施）」を持つだけで、工程の細分化や進捗の可視化がない。

**実装案**：
- 案件ごとに工程（タスク）を定義（工種・開始日・終了日・担当者・進捗率）
- ガントチャートで横断表示
- 案件横断でも表示可能（全社の工程を1画面で）
- 工程の遅延を色で警告
- 担当者の稼働状況も同じ画面で確認

#### B. 作業日報＋稼働時間記録

サクミル/プロワン共通の中核機能。「日報→原価集計→経営レポート」の流れを作る。

**実装案**：
- スマホから日報入力（案件選択、作業内容、作業時間、写真）
- 案件×担当者×日付で稼働時間を蓄積
- QCDSの「実際の現場人件費」に自動反映（人件費単価マスタ × 稼働時間）
- ダッシュボードに「案件ごと/人ごとの稼働時間」を表示

#### C. スケジュール管理（現場・職人）

サクミルの中核機能。

**実装案**：
- カレンダービュー（個人・現場・全社の3切替）
- 案件のキーマイルストーン（着工・検査・引渡など）を表示
- 職人・社員の予定を重ね合わせ表示
- ダブルブッキング検知

#### D. 写真台帳の自動生成

サクミルの売りの一つ。撮影した写真から工事写真台帳PDFを自動生成。

**実装案**：
- 進捗ログでアップロードした写真にメタデータ付与（撮影日時、工種、コメント）
- 工種・撮影日でグルーピング
- 写真台帳PDF出力（公共工事提出用フォーマットも考慮）

#### E. 顧客マスタの強化

現状は `client_name` を案件側に持つだけで、顧客マスタがない。

**実装案**：
- `clients` テーブル新設
- 案件と顧客の関連付け（同じ顧客の過去案件を一覧できる）
- 顧客ごとの取引履歴・売掛金・受注実績の集計
- 顧客窓口担当者の管理

### 4.2 【次優先】Phase 1〜2の境界

#### F. 見積→注文→請求の自動連動

サクミル/プロワンの「見積から請求書への自動変換」。

**実装案**：
- 見積書承認後に「注文書を作成」ボタン → 見積データを引き継いで注文書草案を生成
- 注文書「signed_returned」状態で「請求書を作成」ボタン → 工事完了時に請求書草案を生成
- 部分請求（月次の出来高請求）にも対応

#### G. 通知機能（Slack＋メール）

承認フロー設計書では Phase 2 としていたが、競合は当然のように持っているので Phase 1 後半に前倒し。

**実装案**：
- 承認依頼、ステータス変化、期限アラート、日報リマインドを通知
- 通知先：個人ごとのSlack DM、メール
- 通知のON/OFF設定をユーザーごとに

#### H. グループチャット・案件コメント機能

サクミルが持つ機能。現場と事務所のコミュニケーションを集約。

**実装案**：
- 案件ごとに掲示板的なコメント欄
- @メンションで通知
- 写真・ファイル添付
- 既読管理

### 4.3 【中優先】Phase 2

#### I. モバイル最適化の強化

PWAでレスポンシブ対応はする予定だが、サクミル/プロワン水準には届かない。

**実装案**：
- カメラ起動→撮影→アップロードを1タップで完結
- GPS位置情報の自動付与（撮影現場の特定）
- オフライン入力→オンライン時に同期
- ホーム画面追加プロモート

#### J. 工事写真の自動分類（AI）

Geminiを使って撮影写真を工種別に自動分類するAI機能。

**実装案**：
- 写真アップロード時にVisionで「これは内装工事の写真です」のように自動タグ付け
- 写真台帳のグルーピングを自動化

#### K. 会計ソフト連携

freee / マネーフォワード への請求書連携。

#### L. ルート最適化（プロワン由来）

複数現場を回る場合のルート最適化。クラップさんの業態（建築改修中心）では優先度低。

### 4.4 【低優先】Phase 3以降

- AI電話応対（プロワン由来、優先度低）
- 補助金IT導入対応
- ホワイトラベル化（他社にも提供するなら）

## 5. クラップさんが「現行Excel以上」を実現するための4つの観点

単に競合機能を取り込むだけでなく、クラップさんの業務にフィットする観点で整理：

### 5.1 「数値の正確性」をExcel以上に

現行Excel：手入力ミス、コピペミス、計算式の壊れ、シート間リンク切れが起こる。

CM-v3で実現すべきこと：
- 入力時の即時バリデーション（金額の桁、日付の整合性）
- 計算根拠の透明化（ⓘホバーで「この数値はどう計算されたか」が見える）
- 編集履歴で過去の数値変更が追える
- 楽観的ロックで同時編集の事故を防ぐ

### 5.2 「情報の流れ」をExcel以上に

現行Excel：見積書 → 注文書 → 請求書がそれぞれ別ファイル、転記が手作業。

CM-v3で実現すべきこと：
- 見積から注文、注文から請求が自動引き継ぎ
- 業者見積（外部）→ QCDS（内部）→ 見積書（顧客向け）が一気通貫
- 案件・顧客・業者の3軸で過去データを横断検索

### 5.3 「現場と事務所の距離」をゼロに

現行Excel：現場担当者が事務所に戻らないと入力できない、紙の日報、写真は別管理。

CM-v3で実現すべきこと：
- スマホから現場入力（日報、写真、進捗）
- リアルタイム共有（事務所が即座に把握）
- 「現場で完結」できるUI（再入力させない）

### 5.4 「意思決定」をExcel以上に

現行Excel：集計しないと全体が見えない、月次決算の翌月にしか経営状況がわからない。

CM-v3で実現すべきこと：
- リアルタイム経営ダッシュボード
- 案件単位の損益が確定前から見える（実行予算 vs 取決見通 vs 精算）
- 利益率ランキング、期限超過アラート、キャッシュフロー予測

## 6. 推奨：CM-v3 設計への追加事項まとめ

設計書に以下の追補を追加することを推奨：

### 設計書追補③：施工管理機能（工程表・日報・スケジュール）

- **工程表（ガントチャート）** — 案件単位＋全社横断
- **作業日報** — 案件×担当者×日付、スマホ入力
- **稼働時間集計** — 日報からQCDS人件費・経営ダッシュボードへ連動
- **スケジュール管理** — カレンダービュー、ダブルブッキング検知
- **写真台帳自動生成** — 工事写真台帳PDF出力

### 設計書追補④：顧客・取引先管理機能の強化

- **顧客マスタ** — 案件と独立、過去取引集計
- **顧客窓口担当者** — 客先担当を構造化
- **取引履歴** — 顧客ごとの売上・受注・売掛金

### 設計書追補⑤：帳票の自動連動

- **見積→注文→請求の引き継ぎ** — 草案自動生成
- **部分請求** — 出来高による分割請求
- **入金消込** — 請求書と入金の紐付け

### 設計書改訂①：通知機能を Phase 1 後半に前倒し

承認フロー設計書で Phase 2 としていた Slack/メール通知を Phase 1 に組み込む。

### 設計書改訂②：モバイル対応の格上げ

「PWAレスポンシブのみ」から「スマホファースト UIの主要画面追加」へ。
特に日報入力・写真アップロード・進捗確認はスマホで完結するように設計。
