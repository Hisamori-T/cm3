## 0. このドキュメントの位置づけ

競合のプロワン・サクミルの機能スクリーンショットをもとに、Construction Manager v3 に追加すべき機能の具体的な要件を定義する。見積書・QCDS・承認フローはすでにほぼ完成しているため、**未実装の施工管理系機能を中心に記述する**。

## 1. スケジュール・作業管理（カレンダー強化）

### 1.1 現状
- カレンダーページ（`/calendar`）: スケジュールイベントの登録・閲覧のみ
- 日報ページ（`/daily-report`）: 日報の入力・タイムライン表示のみ
- 両者は完全に独立しており、連携なし

### 1.2 目標
**「誰がいつどこの現場にいるか」を1画面で把握できるカレンダー**

サクミルの「スケジュール・作業管理」に相当する機能。

### 1.3 カレンダー表示の4カテゴリ

| カテゴリ | 色 | データソース | 内容 |
|---|---|---|---|
| イベント | 種別ごと（打合せ:紫/マイルストーン:赤等） | `schedule_events`テーブル | 打合せ・業者訪問・マイルストーン |
| 日報 | 緑（#16a34a） | `daily_reports` + `daily_report_entries` | 実際の作業記録・現場名 |
| 担当者スケジュール | シアン（#0891b2） | `schedule_events`（event_type=site_visit） | 誰がどの現場に行く予定か |
| その他 | グレー（#6b7280） | `schedule_events`（event_type=personal） | 個人予定・有給等 |

### 1.4 カレンダーセル表示

各日付セルに以下を表示（最大3件、超過は「+N件」）：
- 現場訪問（担当者スケジュール）：シアン背景・「現場名 / 担当者名」
- 日報（提出済み）：緑背景・「プロジェクト名」
- 打合せ・業者訪問：種別ごとの色・タイトル

### 1.5 日付タップ → 日詳細パネル

カレンダーの任意の日付をクリック → 右側（またはモーダル）に日詳細パネルが表示される。

パネルに表示する内容：
1. **その日のイベント一覧**：時刻・タイトル・種別
2. **その日の日報一覧**：誰が・どの現場で・何時間
3. **アクションボタン**：
   - `+ イベント追加`（クリックするとその日付が自動セットされたイベント作成フォームが展開）
   - `+ 日報を書く`（クリックするとその日付が自動セットされた日報入力フォームが展開）

### 1.6 日報入力フォーム（カレンダーから）

日詳細パネル内または独立モーダルとして表示。日付は自動設定（変更可）。

入力項目：
- 日付（自動設定）
- 天気（☀曇☁雨🌧雪❄）
- 案件（ドロップダウン選択）
- 作業内容（テキスト）
- 開始時刻 / 終了時刻
- 休憩（分）
- メモ

複数案件を1日報に追加可（「+ 案件を追加」ボタン）。

### 1.7 「担当者の現場スケジュール入力」

**現場訪問予定の登録機能（イベント登録の拡張）：**

「+ 現場スケジュール追加」ボタン → 以下を入力：
- 担当者（ユーザー選択、複数可）
- 案件（案件ドロップダウン）
- 日付 / 時刻
- メモ

内部的には `ScheduleEvent`（event_type=site_visit）に `project_id` と `attendees` を設定して登録。

**管理者は全員分、一般社員は自分分のみ表示（閲覧は全員可）。**

## 2. ガントチャート（工程表）

### 2.1 現状
- ガントチャートページ（`/projects/[id]/gantt`）は存在するが未完成
- `project_tasks` テーブルは設計済み

### 2.2 目標
CrowdLog / サクミルの「ガントチャート」に相当する機能。

**表示軸の切替：**
- **プロジェクト軸**：案件ごとに工程（タスク）をガントバーで表示
- **メンバー軸**：担当者ごとに稼働案件を一覧（誰がいつどの現場に）

**機能要件：**
1. 案件ごとの工程（タスク）定義：タスク名・開始日・終了日・担当者・進捗率
2. ガントバーのドラッグ＆ドロップで日程変更
3. 遅延タスクを赤色でハイライト（今日が終了日を超えかつ進捗率<100%）
4. 全案件横断ビュー（全社の工程を1画面で）
5. マイルストーン表示（菱形マーカー）
6. 依存関係の矢印（Phase 2）

### 2.3 DB（参考：project_tasks テーブル）

```
project_tasks:
  id, project_id, parent_task_id, name
  start_date, end_date, progress_pct
  assignee_id, is_milestone
  depends_on_ids (JSON配列)
```

## 3. 写真・ファイル管理強化

### 3.1 現状
- 進捗ページで写真・図面のアップロード/表示は実装済み
- タグ付けなし、フォルダ管理なし

### 3.2 目標
サクミルの「写真・ファイル管理」に相当する機能。

**機能要件：**
1. **フォルダ管理**：「施工前」「施工中」「施工後」「図面」等のフォルダ分類
2. **タグ付け**：工種・撮影フェーズのタグ自由設定
3. **写真台帳PDF出力**：タグ・日付でグルーピングした写真台帳を自動生成（WeasyPrintで出力）
4. **スマホから直接撮影→アップロード**：`<input type="file" accept="image/*" capture="environment">`
5. **GPS情報取得**（Phase 2）：撮影位置を自動付与

## 4. 発注管理

### 4.1 現状
- `発注書`タブ（`/projects/[id]/order`）は実装済み
- 見積データからの自動入力ボタンも実装済み

### 4.2 目標（Phase 2 相当）

**追加要件：**
1. **発注書一覧**：全案件横断で発注書を一覧表示（ステータス・支払状況・業者名）
2. **ステータス管理**：未発注/発注済/納品済/支払済
3. **支払いカレンダー**：支払期日をカレンダービューに表示

## 5. 請求・入金管理

### 5.1 現状
- 請求書ページ（`/projects/[id]/invoice`）は実装済み
- 入金管理は未実装

### 5.2 目標（Phase 1-C 相当）

**追加要件：**

1. **入金管理**（最優先）：
   - 請求書に「入金日」「入金額」「残高」フィールド追加
   - 請求書ステータス：`未請求 → 請求済 → 一部入金 → 入金完了`
   - 入金消込処理（複数回の入金に対応）

2. **入金アラーム**：
   - 請求日から30日超過で未入金の案件をダッシュボードで警告表示
   - 対象案件の一覧ウィジェット「未入金・期限超過」

3. **売掛金サマリー**：
   - ダッシュボードに「今月請求予定 / 入金待ち / 期限超過」の3カードを追加

## 9. 技術メモ

### カレンダー日報連携
- バックエンド：`GET /api/v1/daily-reports?from_date=&to_date=` は実装済み
- フロントエンド：カレンダー月変更時に日報データも並行取得
- `DailyReportEntry.project_name` をAPIレスポンスに追加する必要あり（現状は`project_id`のみ）

### 担当者スケジュール
- `schedule_events` の `event_type=site_visit` かつ `project_id` が設定されているものを「担当者スケジュール」として扱う
- `ScheduleEventAttendee` で複数の担当者を紐づけ可能
- イベント作成時に `attendee_user_ids` を指定する

### 入金管理
- 既存の `invoices` テーブルに `received_at` / `received_amount` フィールドを追加（Alembicマイグレーション必要）
- または `invoice_payments` テーブルを別途作成（複数回入金に対応）
