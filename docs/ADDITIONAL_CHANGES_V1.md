## 2. QCDS への反映

### 2.1 「QCDSに反映」ボタンの動作

```
業者見積タブ → 版1（清水ウェルディング）→ [QCDSに反映]
  ↓
ダイアログ表示:
  ┌─────────────────────────────────┐
  │ QCDSに反映                      │
  │                                 │
  │ カテゴリーを選択:               │
  │ ◉ 外注業者                     │
  │ ○ 資材業者                     │
  │ ○ その他                       │
  │                                 │
  │ 反映内容:                       │
  │   業者名: 清水ウェルディング    │
  │   予算: ¥2,800,000              │
  │                                 │
  │ [反映する] [キャンセル]         │
  └─────────────────────────────────┘
  ↓
QCDS の「A. 直接工事」の該当カテゴリー列に 1 行追加:
  外注業者: 清水ウェルディング ¥2,800,000
  （または）
  資材業者: 清水ウェルディング ¥2,800,000
  （または）
  その他: 清水ウェルディング ¥2,800,000
```

### 2.2 API

```python
POST /api/v1/projects/{project_id}/qcds/reflect-from-version
{
    "version_id": "uuid",
    "category": "subcontractor" | "material" | "other",
    # subcontractor = 外注業者（A-1列）
    # material = 資材業者（A-2列）
    # other = その他（A-3列）
}

# 処理:
# 1. version の合計金額を取得
# 2. QCDS の指定セクション・カテゴリーに 1 行追加
#    item_name = version.vendor_name
#    amount = version.total_amount
# 3. 業者見積版に「QCDS反映済み」フラグを設定
```

## 3. 顧客見積への反映

### 3.1 「顧客見積に反映」ボタンの動作

```
業者見積タブ → 版1（清水ウェルディング）→ [顧客見積に反映]
  ↓
ダイアログ表示:
  ┌──────────────────────────────────────────┐
  │ 顧客見積に反映                            │
  │                                          │
  │ 掛率: [1.2 ▼]                            │
  │ ※ 版の全体掛率が適用されます             │
  │                                          │
  │ 大項目:                                  │
  │ ◉ 新しい大項目を作成                     │
  │   大項目名: [外壁工事          ]          │
  │ ○ 既存の大項目に追加                     │
  │   [A. 共通仮設工事 ▼]                    │
  │                                          │
  │ [反映する] [キャンセル]                   │
  └──────────────────────────────────────────┘
  ↓
顧客見積に品目が追加される:
  - 掛率が適用された販売単価で追加
  - cost_price = 業者見積の原価単価（参考情報）
  - source_type = 'scan'
  - 追加後は顧客見積内で自由に加筆・修正が可能
```

### 3.2 API

```python
POST /api/v1/projects/{project_id}/quote/{quote_id}/reflect-from-version
{
    "version_id": "uuid",
    "markup_rate": 1.2,
    "target_section": {
        "type": "new" | "existing",
        "section_name": "外壁工事",      # type=new の場合
        "section_id": "uuid"             # type=existing の場合
    }
}

# 処理:
# 1. type=new なら quote_sections に新しい大項目を作成
# 2. version の全品目を quote_items にコピー
#    - unit_price = cost_price × markup_rate
#    - amount = unit_price × quantity
#    - cost_price = version_item.unit_price（参考情報）
#    - source_type = 'scan'
# 3. 追加後の品目は顧客見積内で自由に編集可能
```

### 3.3 顧客見積内での編集

```
顧客見積に反映されたデータは「入力補助」として使用される。
反映後は顧客見積内で以下が自由にできる:
  - 品名の変更
  - 仕様の変更
  - 単価の変更
  - 数量の変更
  - 行の追加・削除
  - 大項目の上下移動（row_no スワップ）

つまり、業者見積の版と顧客見積の品目は「コピー」であり、
反映後のリンクは維持しない（独立したデータになる）。
```

## 4. 業者マスタへの自動転記

### 4.1 スキャン時の自動転記

```
スキャン完了時:
  1. 業者名が業者マスタに存在するか確認
     → 存在しない: 新規登録
     → 存在する: 既存レコードを使用
  2. スキャン結果の品目・単価を VendorPriceHistory に追加
     → 「いつ」「どの案件で」「何を」「いくらで」を記録
```

### 4.2 業者マスタからの転記

```
案件詳細 → 業者見積タブ
  ↓
「業者マスタから転記」ボタン
  ↓
業者を選択:
  ┌─────────────────────────────────┐
  │ 業者マスタから転記              │
  │                                 │
  │ 業者を検索: [清水ウ___         ]│
  │                                 │
  │ 清水ウェルディング              │
  │   過去案件: 5件                 │
  │   直近単価: H300... ¥189/kg    │
  │                                 │
  │ どの案件のデータを転記？        │
  │ ○ 案件A（2026/03）             │
  │ ◉ 案件B（2026/01）             │
  │                                 │
  │ [転記する] [キャンセル]         │
  └─────────────────────────────────┘
  ↓
選択した案件の業者見積データが版として追加される
  ↓
スキャン後と同じ操作が可能:
  - 「QCDSに反映」ボタン
  - 「顧客見積に反映」ボタン
  - 内容の確認・修正
```

### 4.3 API

```python
# 業者マスタから版を作成
POST /api/v1/projects/{project_id}/quote-versions/create-from-vendor
{
    "vendor_id": "uuid",
    "source_project_id": "uuid"  # どの案件のデータを転記するか
}

# 処理:
# 1. source_project_id の業者見積版（指定 vendor_id）を取得
# 2. 現在の project_id に新しい版としてコピー
# 3. 版名 = vendor_name
# 4. 品目もすべてコピー
# 5. 以降はスキャン後と同じ操作が可能
```
